Tablouri unidimensionale; Vectori

Noțiuni fundamentale despre declararea, alocarea și utilizarea vectorilor în C++

Definiția și declararea vectorului

Un tablou unidimensional (vector) reprezintă o colecție de variabile de același tip, memorate într-o zonă continuă de memorie.

Sintaxa generală:
tipDeDate denumire[DimensiuneFizică];

Exemple de declarări corecte:

int X[10];                 // Un vector capabil să rețină maxim 10 numere întregi
int n, X[10], m, Y[100];   // Declarare multiplă (variabile simple și vectori)

Observații Critice și Comportament Memorie

Alocare Globală vs. Locală

Elementele unui tablou declarat global sunt inițializate automat cu 0. Dacă tabloul este declarat local (în interiorul `main`), elementele vor conține inițial valori aleatorii (reziduuri din memorie).

Indexare nativă de la 0

În limbajul C++, indexarea elementelor se face în mod nativ începând de la poziția 0. Un vector cu N elemente va avea indicii cuprinși între 0 și N - 1.

Referirea unui element

Accesul la o anumită valoare din vector se realizează prin intermediul operatorului de indexare [], care are prioritate maximă în ordinea executării operațiilor.

Exemple de referire: X[0], X[5], sau generalizat X[i] (unde i este o expresie întreagă validă).

Un element referit corect se comportă exact ca o variabilă oarecare din tipul de bază al tabloului:

int X[10];
X[0] = 17;

cout << X[0];     // Afișează valoarea: 17
cout << X[0] + 5; // Efectuează calculul: 22
cout << X[0] / 5; // Diviziune întreagă: 3

Dimensiunea fizică vs. Dimensiunea logică

La declarare se stabilește dimensiunea fizică (numărul maxim de celule rezervate în memorie). În timpul execuției, se utilizează o dimensiune logică, reprezentând numărul curent de elemente populate efectiv.

Parcurgerea unui vector

Reprezintă vizitarea și/sau modificarea fiecărui element într-o ordine bine stabilită (directă sau inversă), folosind o structură repetitivă:

int X[100], n; // n reprezintă numărul de elemente utilizate efectiv
for(int i = 0 ; i < n ; i++) {
    X[i] = 1;  // Inițializăm toate elementele logice cu valoarea 1
}

Operații de Citire și Afișare

Deoarece un vector este un ansamblu de elemente, citirea sau afișarea lui directă (ca o singură entitate obiectuală) nu este permisă conceptual în standardul simplu C++.

Greșeli Frecvente la Examen

cin >> X; → Eroare de sintaxă (compilatorul nu știe câte elemente să citească).

cout << X; → Nu dă eroare, dar va tipări adresa în hexazecimal a primului element din memorie (ex. 0x7ffc9711bcd0).

Implementarea Corectă

Citirea element cu element: 

cin >> n;
for(int i = 0 ; i < n ; i++)
    cin >> X[i];
Afișarea element cu element: 
for(int i = 0 ; i < n ; i++)
    cout << X[i] << " ";

Strategii de abordare: Indexare de la 0 vs. Indexare de la 1

Deși tehnic vectorii încep de la poziția 0, în rezolvările problemelor de la examenul de Bacalaureat se practică adesea ignorarea celulei 0 pentru a mapa indicii direct cu logica enunțului.

Cum procedăm dacă vrem indexare de la 1?

1. Celula X[0] va fi lăsată neutilizată în memorie.

2. Structurile repetitive se vor executa în intervalul convenabil: for(int i = 1 ; i <= n ; i++).

3. Atenție la dimensionare: Dacă enunțul specifică o dimensiune maximă de n ≤ 100 elemente, tabloul va fi declarat obligatoriu de dimensiune cel puțin 101 pentru a evita depășirea de memorie (out of bounds) la accesarea lui X[100].

int X[101]; // Alocă celule de la 0 la 100. Celula X[100] devine astfel accesibilă!

Inițializarea elementelor direct la declarare

Dacă se cunosc valorile de la început, limbajul C++ permite utilizarea listelor de inițializare închise între acolade:

Exemplu de cod Comportament dedus în memorie
int A[] = {10, 20, 30, 40}; Dimensiunea fizică este determinată automat la 4 elemente (de la A[0] la A[3]).
int B[10] = {10, 20, 30, 40}; Primele 4 elemente preiau valorile date, iar restul de 6 elemente rămase sunt inițializate automat cu 0.
int C[10] = {1}; Doar primul element C[0] va fi setat pe valoarea 1. Toate celelalte 9 elemente vor fi automat 0!
