Metoda Backtracking

Teorie completă + probleme de BAC explicate pas cu pas

Introducere intuitivă

Imaginează-ți că ești într-un labirint necunoscut și încerci să găsești ieșirea. Mergi înainte pe o potecă până când dai de un perete. Ce definiție aplici? Nu te blochezi acolo, ci te întorci la ultima intersecție și încerci o altă variantă.

Exact asta face și metoda Backtracking (în traducere: „revenire pe drumul parcurs”). Este o metodă algoritmică de parcurgere sistematică a tuturor soluțiilor posibile ale unei probleme. Ea construiește soluția pas cu pas, iar dacă la un moment dat constată că direcția aleasă nu poate duce la un rezultat corect, abandonează acea ramură și se întoarce înapoi pentru a încerca altceva.

Definiție și Ideea Metodei

Se construiește un vector soluție x[] = (x[1], x[2], ..., x[n]), unde fiecare element x[k] primește valori dintr-o mulțime specifică de elemente disponibile.

Mecanismul de bază la fiecare pas (k):
Alegem o valoare pentru x[k] → Verificăm dacă este validă parțial → Dacă da, continuăm la pasul următor (k+1); dacă nu, încercăm următoarea valoare disponibilă sau revenim la pasul anterior (k-1).

Tipuri de generări clasice la BAC

Pentru a rezolva rapid problemele grilă, trebuie să identifici ce structură combinatorică folosește enunțul:

Tipul generării Regulă principală Exemplu pentru {1, 2, 3}
Permutări Se folosesc TOATE elementele, contează ordinea, cifrele nu se repetă. (1,2,3), (1,3,2), (2,1,3)...
Aranjamente Se folosesc doar k elemente din cele n, contează ordinea. De câte 2: (1,2), (1,3), (2,1)...
Combinări Se folosesc doar k elemente, ORDINEA NU CONTEAZĂ (sunt mereu în ordine strict crescătoare). De câte 2: (1,2), (1,3), (2,3)

Schemă generală (Implementare recursivă)

void back(int k)
{
    for(int i = A; i <= B; i++) // parcurgem valorile posibile
    {
        x[k] = i; // selectăm valoarea

        if(ok(k)) // condiția internă (validare parțială)
        {
            if(solutie(k)) 
                afis(k); // am ajuns la o soluție finală
            else 
                back(k+1); // trecem la pasul următor
        }
    }
}

Termeni importanți

  • Condiții externe: Mulțimea valorilor pe care le poate lua fiecare element x[k] în mod brut (de exemplu: cifre de la 0 la 9).
  • Condiții interne: Filtrele logice care verifică dacă elementul ales păstrează soluția validă (de exemplu: x[k] să fie distinct de elementele anterioare).
  • Soluție finală: Momentul în care vectorul este complet completat și îndeplinește criteriul problemei.

Exercițiul 1 (BAC 2025)

Enunț:

Utilizând metoda backtracking se generează toate permutările elementelor mulțimii ordonate {1, 2, 3, 4, 5, 6}; pentru fiecare permutare, pe primele trei poziții sunt doar valori pare, iar pe ultimele trei poziții sunt doar valori impare. Primele șase soluții generate sunt: (2,4,6,1,3,5), (2,4,6,1,5,3), (2,4,6,3,1,5), (2,4,6,3,5,1), (2,4,6,5,1,3), (2,4,6,5,3,1). Indicați a șaptea soluție generată.

Explicație pas cu pas:

Problema împarte vectorul soluție în două zone distincte: primele 3 poziții conțin numerele pare {2, 4, 6}, iar ultimele 3 poziții conțin numerele impare {1, 3, 5}.

  • Observăm că în primele 6 soluții oferite de enunț, prefixul format din cifrele pare rămâne complet blocat pe valoarea (2, 4, 6).
  • În acest timp, sufixul format din cifre impare rulează prin toate cele 6 permutări posibile ale mulțimii {1, 3, 5}, ajungând la ultima sa variantă lexicografică, adică (5, 3, 1).
  • Odată ce ramura numerelor impare s-a epuizat, algoritmul face backtrack la primele trei poziții pentru a modifica prefixul de numere pare.
  • Următoarea permutare lexicografică a mulțimii {2, 4, 6} după (2, 4, 6) este (2, 6, 4) (cifra 4 crește la următoarea valoare pară disponibilă, adică 6).
  • După stabilirea noului prefix, zona numerelor impare se resetează de la început, reluând prima permutare în ordine lexicografică: (1, 3, 5).

Răspuns corect: (2, 6, 4, 1, 3, 5)

Exercițiul 2 (BAC 2024)

Enunț:

Utilizând metoda backtracking se generează toate numerele formate din cifre distincte, cifre a căror sumă este 6. Primele patru soluții generate sunt, în această ordine: 1023, 1032, 105, 1203. Indicați cea de a șasea soluție generată.

Explicație pas cu pas:

Algoritmul generează numerele în ordine lexicografică strictă (ordonare crescătoare). Să urmărim generarea ramurilor:

  • Soluția 1: 1023 (Suma: 1+0+2+3 = 6)
  • Soluția 2: 1032 (Suma: 1+0+3+2 = 6)
  • Soluția 3: 105 (Suma: 1+0+5 = 6) → s-a încheiat ramura numerelor care încep cu prefixul 10...
  • Algoritmul avansează la următorul prefix posibil, adică 12...
  • Soluția 4: 1203 (Suma: 1+2+0+3 = 6)
  • Soluția 5: După 1203, algoritmul face backtrack la lungimea de 3 cifre cu prefixul 12. Următoarea cifră mai mare decât 0 este 3, obținându-se numărul 123 (Suma: 1+2+3 = 6).
  • Soluția 6: Ramura prefixului 12 fiind complet epuizată, algoritmul trece la prefixul următor, 13.... Prima combinație validă de cifre distincte care adunate dau 6 este 1302 (Suma: 1+3+0+2 = 6).

Răspuns corect: 1302

Exercițiul 3 (BAC 2021)

Enunț:

Se generează toate numerele de 3 cifre formate doar din cifre impare (1, 3, 5, 7, 9), utilizând metoda backtracking. Se cere a șasea soluție generată.

Explicație pas cu pas:

Deoarece enunțul nu menționează că cifrele trebuie să fie distincte, înseamnă că elementele din mulțimea {1, 3, 5, 7, 9} se pot repeta. Generăm numerele crescător, pas cu pas:

  1. Soluția 1: 111
  2. Soluția 2: 113
  3. Soluția 3: 115
  4. Soluția 4: 117
  5. Soluția 5: 119
  6. Soluția 6: După ce ultima cifră a ajuns la valoarea maximă (9), algoritmul face backtrack la poziția a doua. Cifra 1 de pe poziția a doua este înlocuită cu următoarea cifră impară (3), iar ultima poziție se resetează la valoarea de pornire (1). Obținem numărul 131.

Răspuns corect: 131

Concluzie pentru examen

La BAC, problemele de backtracking de la subiectul I nu cer să scrii cod complet, ci să înțelegi ordinea lexicografică și modul în care se modifică elementele de la dreapta la stânga în momentul unei reveniri (backtrack). Regula de aur: Schimbă ultima poziție; dacă s-a epuizat, mergi cu un pas la stânga, mărește valoarea și o ia de la capăt!
