Teoria Grafurilor

Ghidul complet pentru BAC: Grafuri neorientate, orientate, arbori și reprezentări

1. Grafuri Neorientate. Noțiuni de bază

Un graf neorientat este o pereche ordonată de mulțimi G = (V, M), unde:

  • V este o mulțime finită și nevidă de elemente numite vârfuri (sau noduri).
  • M este o mulțime de perechi neordonate de elemente distincte din V, numite muchii. O muchie între nodurile u și v se notează [u, v].

Gradele nodurilor și Teoreme Fundamentale (Foarte importante pentru grilă!):

  • Gradul unui nod v (notat d(v)): reprezintă numărul de muchii care trec prin acel nod.
  • Nod izolat: un nod care are gradul 0 (nu are nicio muchie).
  • Nod terminal: un nod care are gradul 1.
  • Teorema celor două strângeri de mână: Suma gradelor tuturor nodurilor dintr-un graf neorientat este egală cu dublul numărului de muchii:
    ∑ d(v) = 2 • m
  • Consecință directă: În orice graf neorientat, numărul de noduri cu grad impar este întotdeauna un număr par.

2. Tipuri Speciale de Grafuri Neorientate

La BAC vei întâlni des probleme care cer proprietăți ale anumitor structuri standard:

Tip Graf Definiție și Proprietăți la BAC Formulă număr muchii
Graf Complet (Kn) Există muchie între oricare două noduri distincte. Fiecare nod are gradul n - 1. n • (n - 1) / 2
Graf Bipartit Nodurile pot fi împărțite în două mulțimi disjuncte X și Y, astfel încât orice muchie are un capăt în X și celălalt în Y (nu există muchii între noduri din aceeași mulțime). Maxim: p • q
(unde p=|X|, q=|Y|)
Graf Bipartit Complet Un graf bipartit în care fiecare nod din X este legat de toate nodurile din Y. Exact: p • q

Modificări de structură (Diferențe subtile):

  • Graf Parțial: Se păstrează TOATE nodurile, dar se elimină zero sau mai multe muchii.
  • Subgraf: Se elimină una sau mai multe noduri, și automat se șterg toate muchiile care aveau un capăt în nodurile eliminate.

3. Lanțuri, Cicluri și Conexitate

  • Lanț: O succesiune de noduri în care oricare două noduri consecutive sunt adiacente (legate prin muchie).
    • Lanț elementar: conține doar noduri distincte.
    • Lanț simplu: conține doar muchii distincte.
  • Ciclu: Un lanț în care primul nod coincide cu ultimul nod.
    • Ciclu elementar: toate nodurile sunt distincte (cu excepția primului și ultimului).
  • Graf Conex: Un graf în care între oricare două noduri distincte există cel puțin un lanț.
  • Componentă conexă: Un subgraf conex maximal (nu mai poate fi extins prin adăugarea altor noduri fără să-și piardă conexitatea).
  • Număr minim de muchii ca un graf cu n noduri să fie conex: n - 1 muchii.
  • Număr maxim de muchii pe care le poate avea un graf cu n noduri fără a fi conex: Se izolează un nod, iar celelalte n-1 noduri formează un graf complet: (n - 1) • (n - 2) / 2 muchii. Dacă mai adaugi o singură muchie, graful devine obligatoriu conex!

4. Arbori (Subiect extrem de frecvent)

Un arbore este un graf neorientat, conex și aciclic (fără cicluri).

Proprietăți fundamentale ale arborilor cu n noduri:

  1. Are exact n - 1 muchii (Dacă un graf are n noduri și n-1 muchii și este conex, atunci e arbore).
  2. Între oricare două noduri distincte există un lanț elementar unic.
  3. Orice arbore cu cel puțin 2 noduri conține cel puțin două noduri terminale (frunze, cu gradul 1).
  4. Dacă se mai adaugă o muchie oarecare, se formează exact un ciclu elementar.

5. Grafuri Orientate

Un graf orientat este o pereche ordonată de mulțimi G = (V, A), unde legăturile dintre noduri au un sens stabil numit arc. Un arc pornește de la nodul u (extremitate inițială) și ajunge în nodul v (extremitate finală), notat (u, v).

Gradele în grafuri orientate:

  • Gradul exterior (d+(v)): numărul de arce care pleacă din nodul v.
  • Gradul interior (d-(v)): numărul de arce care intră în nodul v.
  • Teoremă: Suma gradelor exterioare este egală cu suma gradelor interioare și este egală cu numărul total de arce (m):
    ∑ d+(v) = ∑ d-(v) = m

Conexitate specifică:

  • Un graf orientat este tare conex dacă pentru oricare două noduri distincte u și v, există atât drum de la u la v, cât și drum de la v la u.
  • Numărul maxim de arce într-un graf orientat cu n noduri fără circuite este n • (n - 1) / 2.

6. Reprezentarea Grafurilor în C++

La problema de cod (subiectul III), va trebui să știi să stochezi graful în memorie.

A. Matricea de adiacență

O matrice pătratică A de dimensiune (n+1) x (n+1), unde:

  • A[i][j] = 1, dacă există muchie/arc între nodul i și nodul j.
  • A[i][j] = 0, în caz contrar.
  • La grafuri neorientate: matricea este simetrică față de diagonala principală (A[i][j] == A[j][i]). Elementele de pe diagonala principală sunt întotdeauna 0.
// Citirea unui graf neorientat cu matrice de adiacență
int n, m, u, v;
int A[100][100];

cin >> n >> m;
for(int i = 1; i <= m; i++) {
    cin >> u >> v;
    A[u][v] = 1;
    A[v][u] = 1; // Doar la grafuri neorientate!
}

B. Liste de adiacență (Eficiente ca memorie)

Se folosește un vector de liste (din STL - std::vector). Economisește spațiu când graful are puține muchii.

#include <vector>
using namespace std;

vector<int> L[100]; // Un vector de vectori
int n, m, u, v;

cin >> n >> m;
for(int i = 1; i <= m; i++) {
    cin >> u >> v;
    L[u].push_back(v);
    L[v].push_back(u); // Doar la grafuri neorientate!
}

7. Algoritmi de Parcurgere (BFS și DFS)

Parcurgerea în lățime (BFS - Breadth-First Search)

Mecanism: Se folosește o coadă (queue). Se vizitează nodul de start, apoi vecinii lui, apoi vecinii vecinilor și așa mai departe (nivel cu nivel).

Aplicație la BAC: Este algoritmul ideal pentru determinarea drumului minim (ca număr de muchii) de la un nod la altul.

Parcurgerea în adâncime (DFS - Depth-First Search)

Mecanism: Se folosește o abordare recursivă (sau o stivă). Se înaintează pe o cale cât mai adânc posibil până când se lovește un nod fără vecini nevizitați, moment în care se întoarce (backtrack).

Aplicație la BAC: Determinarea componentelor conexe, verificarea conexității grafului sau detectarea ciclurilor.
