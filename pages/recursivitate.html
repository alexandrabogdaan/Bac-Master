Recursivitate

Noțiuni fundamentale despre recursivitate în programare

Definiția recursivității

Recursivitatea este o tehnică de programare în care un subprogram se apelează pe sine însuși (direct sau indirect).

Fiecare funcție recursivă trebuie să aibă o condiție de oprire (caz de bază).

Exemple clasice

  • Factorialul unui număr
  • Cel mai mare divizor comun (CMMDC)
  • Suma cifrelor unui număr
  • Șirul lui Fibonacci

Mecanismul Stivei (Stack)

La fiecare autoapel, starea curentă a funcției (variabilele locale, parametrii) este salvată pe stiva de execuție. Când se atinge cazul de bază, stiva începe să se descarce în ordine inversă.

Exemplu: Factorialul unui număr (n!)

Pentru a înțelege diferența dintre abordări, iată cum calculăm factorialul unui număr natural în ambele variante:

Varianta Iterativă (nerecursivă)

int factorial_iterativ(int n) {
    int p = 1;
    for(int i = 1; i <= n; i++)
        p = p * i;
    return p;
}

Varianta Recursivă

int factorial_recursiv(int n) {
    if(n == 0)
        return 1; // cazul de bază
    else
        return n * factorial_recursiv(n - 1);
}

Alt exemplu frecvent: Suma cifrelor unui număr

O problemă clasică de BAC cere calculul sumei cifrelor unui număr natural n folosind o funcție recursivă: 

int suma_cifre(int n) {
    if(n == 0)
        return 0; // Cazul de bază: numărul nu mai are cifre
    else
        return n % 10 + suma_cifre(n / 10); // Ultima cifră + suma restului cifrelor
}

Atenție la Capcanele de la BAC!

  • Stack Overflow (Depășirea stivei): Dacă uiți să pui cazul de bază sau dacă autoapelul nu tinde către cazul de bază (ex: apelul `f(n+1)` în loc de `f(n-1)`), programul se va bloca din cauza umplerii memoriei alocate stivei.
  • Ordinea instrucțiunilor: Instrucțiunile scrise înainte de autoapel se execută la urcarea pe stivă (în ordinea apelurilor), în timp ce instrucțiunile scrise după autoapel se execută la coborârea de pe stivă (în ordine inversă).
