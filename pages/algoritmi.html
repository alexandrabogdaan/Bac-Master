Algoritmi de Bază

Algoritmii clasici pe numere și tablouri, frecvent ceruți la BAC

De ce sunt importanți?

Algoritmii de bază sunt blocurile fundamentale din care sunt construite probleme mai complexe la BAC. Stăpânirea lor (verificarea numerelor prime, determinarea divizorilor, cmmdc/cmmmc, sortare, căutare) este esențială, deoarece apar atât ca subprobleme în exerciții ample, cât și direct, ca întrebări de tip "ce afișează codul?".

Verificarea unui număr prim

Definiție

Un număr natural n > 1 este prim dacă are exact doi divizori naturali: 1 și el însuși. Numărul 1 nu este considerat prim.

Pentru a verifica dacă n este prim, este suficient să căutăm divizori doar până la √n (rădăcina pătrată), nu până la n – aceasta reduce semnificativ timpul de execuție. 

bool estePrim(int n)
{
    if (n < 2)
        return false;

    for (int d = 2; d * d <= n; d++)
        if (n % d == 0)
            return false; // am gasit un divizor -> nu e prim

    return true; // nu am gasit niciun divizor -> e prim
}
Observație: Condiția d * d <= n este echivalentă cu d <= √n, dar evită utilizarea funcției sqrt() și eventualele erori de rotunjire la numere zecimale.

Determinarea divizorilor unui număr

Pentru a afișa toți divizorii unui număr n, parcurgem toate valorile de la 1 la n și verificăm care dintre ele împart exact pe n. 

void afiseazaDivizori(int n)
{
    for (int d = 1; d <= n; d++)
        if (n % d == 0)
            cout << d << " ";
}
Optimizare: Divizorii unui număr apar în perechi (d și n/d). Putem parcurge doar până la √n și afișa ambii divizori din pereche, reducând complexitatea de la O(n) la O(√n): 
void afiseazaDivizoriOptim(int n)
{
    for (int d = 1; d * d <= n; d++)
    {
        if (n % d == 0)
        {
            cout << d << " ";
            if (d != n / d)
                cout << n / d << " ";
        }
    }
}

CMMDC și CMMMC

CMMDC

Cel Mai Mare Divizor Comun a două numere este cel mai mare număr care le împarte exact pe ambele.

CMMMC

Cel Mai Mic Multiplu Comun a două numere este cel mai mic număr divizibil cu ambele.

CMMDC se calculează eficient cu algoritmul lui Euclid, bazat pe proprietatea: cmmdc(a, b) = cmmdc(b, a % b), până când restul devine 0. 

int cmmdc(int a, int b)
{
    while (b != 0)
    {
        int r = a % b;
        a = b;
        b = r;
    }
    return a;
}

CMMMC se obține folosind formula: cmmmc(a, b) = (a * b) / cmmdc(a, b) 

int cmmmc(int a, int b)
{
    return (a * b) / cmmdc(a, b);
}
Observație: Varianta recursivă a algoritmului lui Euclid este la fel de des cerută la BAC: cmmdc(a, b) = (b == 0) ? a : cmmdc(b, a % b)

Numere perfecte

Definiție

Un număr natural este perfect dacă este egal cu suma divizorilor săi proprii (toți divizorii, exceptând numărul însuși). Exemplu: 6 = 1 + 2 + 3. 

bool estePerfect(int n)
{
    int suma = 0;

    for (int d = 1; d < n; d++)
        if (n % d == 0)
            suma += d;

    return suma == n;
}

Șirul lui Fibonacci

Șirul lui Fibonacci este definit astfel: primii doi termeni sunt 1 și 1 (sau 0 și 1, în funcție de convenție), iar fiecare termen următor este suma celor doi termeni anteriori.

Variantă iterativă (recomandată)

int fibonacci(int n)
{
    int a = 1, b = 1, c;

    if (n <= 2)
        return 1;

    for (int i = 3; i <= n; i++)
    {
        c = a + b;
        a = b;
        b = c;
    }
    return b;
}

Variantă recursivă (lentă)

int fibonacci(int n)
{
    if (n <= 2)
        return 1;
    return fibonacci(n-1) 
         + fibonacci(n-2);
}
Observație: Varianta recursivă este elegantă, dar ineficientă pentru valori mari ale lui n, deoarece recalculează de multe ori aceiași termeni (complexitate exponențială). Varianta iterativă este preferată în practică.

Algoritmi de sortare

Sortarea înseamnă rearanjarea elementelor unui tablou în ordine crescătoare sau descrescătoare. La BAC, cel mai frecvent cerut este algoritmul Bubble Sort, datorită simplității sale.

Bubble Sort (Metoda Bulelor)

Se compară perechi de elemente vecine și se interschimbă dacă nu sunt în ordinea corectă. Procesul se repetă până când tabloul este complet sortat (nicio interschimbare la o parcurgere completă). 

void bubbleSort(int v[], int n)
{
    for (int i = 0; i < n - 1; i++)
        for (int j = 0; j < n - 1 - i; j++)
            if (v[j] > v[j + 1])
            {
                int aux = v[j];
                v[j] = v[j + 1];
                v[j + 1] = aux;
            }
}
Observație: La fiecare parcurgere completă (valoarea lui i), cel mai mare element rămas "se ridică" la poziția corectă din dreapta – de aici și denumirea de "bule". Complexitatea este O(n2) în cazul general.

Algoritmi de căutare

Căutarea secvențială (liniară)

Se parcurge tabloul element cu element, comparându-l cu valoarea căutată. Funcționează pe orice tablou, indiferent dacă este sortat sau nu. Complexitate: O(n).

Căutarea binară

Funcționează doar pe tablouri sortate. La fiecare pas se compară elementul din mijloc cu valoarea căutată și se elimină jumătate din interval. Complexitate: O(log n). 

// Cautare secventiala
int cautareSecventiala(int v[], int n, int x)
{
    for (int i = 0; i < n; i++)
        if (v[i] == x)
            return i; // pozitia gasita

    return -1; // nu a fost gasit
}

// Cautare binara (tabloul TREBUIE sa fie sortat crescator)
int cautareBinara(int v[], int st, int dr, int x)
{
    while (st <= dr)
    {
        int mij = (st + dr) / 2;

        if (v[mij] == x)
            return mij;
        else if (v[mij] < x)
            st = mij + 1;
        else
            dr = mij - 1;
    }
    return -1; // nu a fost gasit
}

Rezumat: complexități de timp
